# Папка для SQL скриптов инициализации БД

Файлы в этой папке выполняются **только при первом запуске** PostgreSQL контейнера 
(когда volume `postgres_data` ещё пустой).

## Порядок выполнения

PostgreSQL выполняет `.sql` и `.sh` файлы в **алфавитном порядке**.

## Как импортировать существующие данные

### Вариант 1: Автоматический импорт при первом запуске

1. Экспортируйте данные с локальной БД:
   ```cmd
   pg_dump -U postgres -d takeItDataBase --data-only -F p -f data.sql
   ```

2. Скопируйте файл в эту папку с именем `01_data.sql`:
   ```
   init-db/01_data.sql
   ```

3. Запустите `docker-compose up -d` - данные импортируются автоматически

### Вариант 2: Ручной импорт после запуска

```bash
# Копируем backup на сервер
scp backup.sql user@server:~/take_it_bot/

# Импортируем
./import_db.sh backup.sql
```

## Файлы

- `00_init.sql` - базовая инициализация (пустой)
- `01_data.sql` - ваши данные (создайте сами)
